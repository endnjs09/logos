# Logos v12 Design

The v12 design goal is to evaluate whether Logos orchestration can lift Gemini
Pro High behavior toward stronger coding-agent behavior.

The implementation direction has one important constraint:

```text
Logos is a host-mounted harness, not a standalone AI coding product.
```

## Product Shape

Logos installs onto existing AI coding CLI hosts.

Primary target:

- Gemini CLI

Compatibility and baseline target:

- Codex CLI

Logos provides:

- commands
- prompts
- roles
- rules
- guards
- workflows
- tools
- plugins
- benchmark and evaluation assets

Logos does not provide:

- a web UI
- an IDE
- a standalone agent runtime
- a model serving layer

## Core Hypothesis

Gemini Pro High can perform better on coding tasks when surrounded by:

- explicit planning
- codebase exploration
- gap analysis
- plan review
- context handoff
- controlled tool use
- verification
- retry policy
- benchmark feedback

## Architecture Principle

Structure is complete early. Implementation is incremental.

The repository should contain final extension points even when runtime support is
not yet implemented.

Important extension points:

- `core`
- `targets`
- `plugins`
- `schemas`
- `benchmarks`
- `runs`
- `reports`

## Mode Principle

The user selects the initial mode:

- Low
- Middle
- High

Logos may recommend escalation when evidence shows the selected mode is too low.

Low should avoid unnecessary process. High should include planning, context
handoff, verification, and review.

## Measurement Principle

Logos must measure improvement. A successful implementation is not just one that
finishes tasks, but one that can show:

- Gemini baseline behavior
- Gemini + Logos behavior
- Codex baseline behavior
- differences between conditions
- failure reasons
- retry counts
- scope violations
- verification quality

## Documentation Principle

Every agent-facing Markdown asset must follow
`docs/reference/markdown-authoring.md`.

Every architectural choice that changes project shape should be recorded under
`docs/decisions/`.
